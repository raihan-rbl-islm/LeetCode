# 1071. Greatest Common Divisor of Strings

**Difficulty:** Easy  
**Topic:** String, Math, Euclidean Algorithm  
**Link:** [LeetCode 1071](https://leetcode.com/problems/greatest-common-divisor-of-strings/)

## Intuition

This problem relies on a mathematical property of strings. If a string $S$ divides both `str1` and `str2`, then `str1` and `str2` are essentially multiples of $S$. This implies that the order of concatenation should not matter: `str1 + str2` must equal `str2 + str1`.

If this condition holds, the greatest common divisor string will have a length equal to the greatest common divisor (GCD) of the lengths of `str1` and `str2`. We can return the prefix of `str1` of that length.

## Complexity

* **Time Complexity:** $O(N + M)$ — The string concatenation and comparison dominate the runtime.
* **Space Complexity:** $O(N + M)$ — Temporary space is required to store the concatenated strings for comparison.

## Code (C++)

```cpp
class Solution {
public:
    string gcdOfStrings(string str1, string str2) {
        // If the combined strings in different orders are not equal, 
        // no common divisor string exists.
        if (str1 + str2 != str2 + str1) return "";
        
        // The length of the GCD string is the GCD of the lengths of the two strings.
        // __gcd is a built-in function in C++ algorithm headers (standard in CP).
        return str1.substr(0, __gcd(str1.size(), str2.size()));
    }
};
